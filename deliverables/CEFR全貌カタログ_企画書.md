# CEFRの全貌を、学習者の手に

### ── 発語内行為・受容・仲介の三カタログによる再設計 ──

**版**: グランドデザイン v1.1（2026年7月）
**位置づけ**: 「場面×CEFRレベル：表現カタログ」提案書の改訂版。問題意識を継承し、解決の設計を刷新する。
**v1.1補正**: v1の「五つの塊」の勘定は、モードで切った塊とスキームで切った塊を混在させており、産出モノローグ等194件がどの塊にも属さず、一部56件が二重に数えられていた。全1,224件を単一の手続きで完全分割し（機械検証済み、`data/block_partition_1224.json`）、**産出・談話構築を第四の縦カタログとして立てて四本柱＋横串に補正**した。

## 1. この企画が解こうとしている問題

CEFR（ヨーロッパ言語共通参照枠）は、世界の言語教育の共通言語になった。英検もTOEICも、いまや自らのスコアをCEFRのA1〜C2に対応づけて語る。だが肝心の学習者にとって、CEFRは依然として**抽象的なチェックリスト**のままである。「B1：身近な話題で意見を述べられる」と言われても、では実際に何をどう言えばB1なのか、その具体は誰も渡してくれない。

CEFRの本体である「能力記述文（descriptor）」は、意図的に**言い方を規定しない**抽象度で書かれている。これは尺度としては正しい設計だが、学習者の側には「できるべきこと」の輪郭だけが与えられ、「では何と言うのか」という具体との間に、埋められない谷が残る。

**本企画の核心的な問題意識は、旧提案書から一貫している** ── CEFRの全体像が、学習者に抽象的チェックリストとしてしか渡っていない。その空白を、具体で埋める。

変わったのは解決の設計である。旧構想は、この谷を「場面」で埋めようとした（レストランで頼んでいないものが出てきたとき、等の場面ごとにA1〜C2の言い方を並べる）。だが後述の精査を経て、**「場面」はキーにならない**ことがわかった。CEFRの構造そのものが、別の骨格を要求していた。

## 2. 発見 ── 全記述文を精査して見えた、六つの異なる塊

CEFR増補版（2020）の全能力記述文を悉皆で精査した。手話を除く音声言語の実記述文は **1,224件**。これを一件ずつ読み、性質で仕分けると、CEFRは均質な梯子ではなく、**性質の根本的に異なる六つの塊**でできていることが明らかになった（全件がちょうど一つの塊に属する完全分割。判定手続き：言語行為論的篩の判定済み244件はその判定に従い「やり取り」へ、残りをスキーム（能力→how well／方略／複言語）、次いでモード（受容／仲介／産出）で割り当てる）。

### 塊①：やり取りの発語内行為（母集団306記述文、うち純粋な「できる」170 → 25種）

「苦情を言う」「食事を注文する」「予約を取る」── 言うことによって何かを遂行する行為。言語行為論（オースティン／サール）でいう発語内行為である。これらは単一の発話や短いやりとりを実例として示せる。学習者が本当に欲しい「じゃあ何て言うの」に、まっすぐ開ける。

やり取り（interaction）に属する記述文は306件。うち発語内行為の候補として篩にかけたのは244件、精査の末に採用したのは **170記述文**。そしてそれらを発語内行為として束ね直すと、**わずか25種**に収斂した。人が外国語で汎用的に遂行すべき発語内行為の型は、この程度の器に収まる。（除外74件は理由付きで台帳に保全。残る62件は全般スケール・会議・電話・対話者理解など単一行為に還元しにくいやり取りで、篩の対象外として同じ塊に帳簿上留めている。）

### 塊②：産出・談話構築（132記述文）── v1.1で立てた第四の柱

まとまった独話（経験談の語り・情報提供・論拠提示）、作文（創作・レポート・エッセイ）、聴衆向け講演、公共アナウンス。**一人で、まとまった長さの談話を構築する**営みであり、単一の発話で実例を示せる発語内行為（塊①）とは別種、受容でも仲介でもない。v1ではこの132件がどの塊にも明示的に置かれていなかった。語り・描写・論証・プレゼンという、学習の実感に近い大きな領域である。

### 塊③：受容（197記述文）

「聞いて理解できる」「読んで理解できる」。これは発語内行為ではない ── 理解は行為の前提であって、遂行ではない。だから①の25種からは原理的に外れる。しかし独立した領域として厳然と存在し、規模も大きい。

### 塊④：仲介（251記述文 ── 最大領域）

CEFR2020の**最大の追加であり、最多の領域**。元の2001年版では狭く「翻訳・通訳」を指すだけだったが、2020年版で大幅に拡張され、初めて実体ある記述文群になった。中身はテクストの仲介（要約・翻訳・データ説明）、概念の仲介（話し合いを促す・意味を共に構築する・グループを運営する）、コミュニケーションの仲介（異文化・対立の橋渡し）の三層（仲介方略はスキーム基準で塊⑥へ）。

①が「一人の話者が行う発語内行為」なら、仲介は「**複数の当事者の間で、理解や意味を成立させる営み**」。行為の主体も目的も根本的に異なる。

### 塊⑤：how well ＝質の軸（182記述文 → 13スケール）

文法的正確さ・語彙の範囲・語彙の使いこなし・音韻・流暢さ・一貫性/結束性・叙述の正確さ・柔軟さ・**社会言語的適切さ**・話題の展開・発言権。これらは縦の目録ではない。①〜④の**すべての行為に横断的にかかる評価軸**である。「苦情を言う」を、文法・語彙・流暢さ・適切さの各軸で測る。質の物差しは行為に依存せず共通、というのがCEFRの設計思想である。

### 塊⑥：方略（104）と複言語・複文化（52）

方略は「どうやって（補償する・言い換える・手がかりから推測する）」── 行為でも質でもなく、手立て。複言語・複文化は「複数言語を横断して立ち回る」力。数は少ないが、性質は他のどれとも違う。

## 3. グランドデザイン ── 四本の縦カタログ＋一本の横串

六つの塊は、そのまま学習者に渡すべきカタログの構造を規定する。

```
                    ┌──────────────────────────────────────────────┐
    横串 →  how well │  文法・語彙・流暢さ・社会言語的適切さ … （13軸）        │
                    └──────────────────────────────────────────────┘
                          ↓ すべての縦カタログに刺さる ↓
    ┌──────────────┬──────────────┬──────────────┬──────────────┐
    │  第1カタログ   │  第2カタログ   │  第3カタログ   │  第4カタログ   │
    │  発語内行為    │  産出・談話構築 │  受容         │  仲介         │
    │ 「やり取りする」│ 「語る・書き上げる」│ 「聞く・読む」 │ 「橋渡しする」 │
    │ 遂行できることの │ 構築できることの │ わかることの   │ 助けることの   │
    │ 目録           │ 目録          │ 目録          │ 目録          │
    │ 25種／170件    │ 132件         │ 197件         │ 251件         │
    └──────────────┴──────────────┴──────────────┴──────────────┘
    （方略・複言語＝当面は各カタログの付随注記、後に独立小カタログを検討）
```

### 第1カタログ：発語内行為 ──「やり取りする」

25の発語内行為を、サールの四類型で整理した。

- **交話・儀礼（2行為）**：挨拶・別れ・安否／会話の開始・維持
- **表出（5行為）**：意見・見解の表明／感情の表出／感謝・詫び・祝意／好悪の表明／理解状態の表明
- **主張（7行為）**：事実情報の授受／質問と応答／問題・事情の説明／面接での質問と応答／伝言の授受／自己に関する情報提供／経験・出来事の叙述
- **指示（11行為）**：明確化・繰り返しの要求／取引（購入・注文）／苦情・クレーム／提案・誘い・計画の相談／依頼・要求／問い合わせ／道案内／助言／同意・不同意／応募・出願／指示への応答

指示型が最多（11行為・61件）。学習者が遂行すべき発語内行為の重心は「相手に何かをさせる／情報を得る」指示型にある。なお**宣言型（declaration：任命・宣告など）はゼロ** ── 学習者に制度的権限がないため、篩の結果として自然に脱落した。

### 第2カタログ：産出・談話構築 ──「語る・書き上げる」

一人でまとまった長さの談話を構築する営みの目録。経験談の語り、描写、情報のまとまった提供、論拠の提示、創作・レポート・エッセイ、聴衆向けの講演。発語内行為が「単一の発話で遂行できる」のに対し、こちらは**談話としての組み立て（順序・展開・結束）そのものが課題**になる。第1カタログの検証で見えた叙述系の梯子（種類→度合い→意味づけ）は、この柱で本格的に展開する。見せ方は第1カタログと異なり、談話の構造（型）とレベル別の実例テクストが対になる。

### 第3カタログ：受容 ──「聞く・読む」

理解の達成の目録。産出ではないため、見せ方は第1カタログと異なる（「この入力を理解できるか」を軸とし、そこから続く応答を実例として開く）。**日本の学習者にとっては、訳読が最も鍛えた領域** ── 「すでにできること」を可視化し、自信の土台とする役割を担う。

### 第4カタログ：仲介 ──「橋渡しする」

他者の理解を助ける営みの目録。第1カタログと双璧をなす柱。要約・言い換え・意味の共同構築・異文化の橋渡し。CEFR2020が最も力を入れて記述した、最も新しく最も大きな領域。

### 横串：how well ── 各セルに埋め込む

13の質の軸は、独立したカタログにはしない。**各can doセルの中に、その行為で決定的に効く軸だけを埋め込む**。13軸は共通に用意されるが、どの軸が効くかは行為ごとに違う。「電話番号を尋ねる」で社会言語的適切さが問われる度合いは小さいが、「苦情を言う」「不同意を表明する」では、適切さの失敗が行為そのものの失敗（苦情が非難になる）に直結する。

### 方略・複言語をどう置くか ── 当面は付随注記、独立化は保留

塊⑥の方略（104件）と複言語・複文化（52件）は、四本の縦カタログのどれにも、横串のhow wellにも、すっきりとは収まらない。だが、だからといって直ちに第五・第六の縦カタログを立てるのは早計である。理由は二つある。第一に、両者は**それ自体が単独で立つ営みではなく、他の営みに寄生して初めて働く**。方略は「どうやって」であり、補償・言い換え・手がかりからの推測は、つねに何らかの行為・産出・受容・仲介を遂行する最中に発動される ── 「言い換えだけを行う」という活動は存在しない。複言語・複文化も同様に、具体的なやりとりの中で複数言語を横断して立ち回る力であって、独立した達成目標として学習者に手渡す形が、現時点では像を結ばない。第二に、規模と成熟度の問題がある。四本の縦カタログの範型がまだ確立していない段階で、性質の異なる小領域を同時に設計に組み込むと、器の形が定まらないうちに複雑さだけが増す。したがって当面は、**方略・複言語を各縦カタログのセルに付随する注記として散らし**（たとえば「苦情・クレーム」のセルに、語彙が出てこないときの補償方略を添える）、縦カタログの範型が固まった後に、独立した小カタログとして立てる価値があるかを改めて判断する。この保留は、両者を軽視するのではなく、**寄生的な性格を持つ領域を、宿主が定まる前に無理に自立させない**という設計判断である。

## 4. 日本の学習者にとっての含意

このグランドデザインは、日本の英語学習者の**学習観そのものを転換する**力を持つ。

### CEFRは「梯子」ではなく「二つの直交する軸」である

多くの学習者は、CEFRをA1からC2への一次元の梯子 ── 上るほど難しいことができる尺度 ── として受け取っている。TOEICのスコアと同じ「高さ」の指標だと。しかし精査が示すのは、CEFRが本当は**二つの直交する軸**でできているということだ。何ができるか（発語内行為・受容・仲介）と、どれくらいうまく（13の質の軸）。

### 上級とは「新しいことができる」ではなく「同じことをより適切に」

純粋な発語内行為は初中級（A2〜B1）に極端に偏り、**C1以上でcan doとして新たに増える行為はほぼ無い**。「苦情を言う」はA2の「何かがおかしいと指摘する」に始まり、B1「苦情を言う」、B2「問題を説明し譲歩を求める」、B2+「補償を要求し交渉する」と、**同じ一つの発語内行為が、レベルを上げながら精緻化していく**。上級とは、A2で既に射程に入った行為を、より適切に遂行できるようになる段階にすぎない。

この像は学習者を解放する。「まだB1だから上のことができない」ではなく、「遂行すべき行為の目録は25個で、その大半はもう自分の射程に入っている。あとはそれを場に合わせてどう遂行するかだ」と。

### 日本の教育の死角 ── 指示型行為と社会言語的適切さ

日本の英語教育が鍛えてきたのは、受容（訳読）と、形式的正確さ（文法・語彙、試験で離散的に測られる質）である。一方、篩に残る発語内行為の核 ── とりわけ**指示型（苦情・依頼・不同意）と、そこで決定的に効く社会言語的適切さ** ── は、系統的に手薄だった。

そしてこの指示型こそ、**日本語からの語用論的転移が最も強く干渉する**領域である。依頼・苦情・不同意は、言語間で適切性条件（felicity condition）が食い違う。日本語の敬語体系や間接性をそのまま持ち込むと、直接的すぎて非難になるか、婉曲すぎて要求が伝わらない。日本語話者にとっての真の課題は「語を知らない」ことではなく、この**語用論的なズレ**にある。

## 5. 実装ロードマップ

### 方針：第1カタログを範型として作り切り、型を第2・第3へ展開する

四本のカタログは性質が異なる。まず第1カタログ（発語内行為）で「**行為 × レベル × 実例（exponent）× L1語用論注意**」というセルの作り方を完成形まで固め、範型を確立する。その型を産出・受容・仲介へ展開すれば、器の形がぶれない（産出・談話構築は談話の組み立てが課題となるため、型の翻案が最も大きい柱になる見込み）。

### なぜ生成AIが要るのか

各セルに、レベル相応の語彙・機能・社会言語的配慮を守った実例発話を複数、そして日本語話者が転移で外す地点を、行為ごとに書く。これを25行為×レベル、さらに産出・受容・仲介まで含めれば数百セル ── 人間だけでは非現実的な作業量である。CEFRのCan-Doは意図的に言い方を規定しない抽象度で書かれており、その抽象と学習者が欲しい具体との間の谷を、**構造は決まっているが件数が膨大で各件が判断を要する**この谷を埋めるのは、まさに生成AIが最も得意とする仕事である。

ただし品質管理が要諦となる。生成した実例が本当にそのレベルか（語彙難易度・CEFRプロファイルとの照合）、L1注意が実際の語用論的転移を正しく突いているか ── これらの検証機構をカタログ生成とセットで設計する。

### 段階

1. **範型確立**：第1カタログの1行為（日本語話者に最も難しい「苦情・クレーム」を推奨）で、レベル別exponent＋L1語用論注意の完成形を作る。
2. **第1カタログ完成**：25行為すべてを範型に沿って埋める。生成AI＋検証機構。
3. **第2〜第4展開**：産出・談話構築、受容、仲介へ型を展開。見せ方の差異（産出＝談話の構築、受容＝理解の達成、仲介＝橋渡しの営み）を吸収する。
4. **横串の統合**：13のhow well軸を各セルに接続。とりわけ社会言語的適切さを、指示型行為で厚く。
5. **学習者への提示形**：分析台帳（作り手の道具）を、体験できる一枚のビューへ翻訳する。25行為を一望し、タップで実例の変化とL1注意が開くインタラクティブな図。

## 6. この企画の独自性

**CEFR全記述文の、具体例カタログ。これは世界にまだ存在しない。**

CEFRのCan-Doリストは各所にある。フレーズ集も無数にある。だが、CEFRの全記述文を発語内行為・産出・受容・仲介に構造化し、各々にレベル別の実例と、特定母語（日本語）話者のための語用論的注意を貼った具体例カタログは、存在しない。理由は単純で、人間だけでは作れなかったからである。生成AIによって初めて現実的になった。

そして大方針は、一貫してこうである ── **CEFRの全貌を、抽象的チェックリストのままにせず、具体とともに、学習者の手に渡す。**
